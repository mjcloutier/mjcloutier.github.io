Setting up a development blog with Ghost

Share this on Twitter, Facebook or Google+.
Published 27 Jan 2015
//

As developers we are encouraged to maintain a blog to share the successes and struggles of our latest discoveries. I figure the first post of this blog will give you the why and how I setup a ghost blog.

Alert: All tutorials on this blog will assume you are using an OSX development environment. It is much more enjoyable and highly recommended!

Why Ghost?

In my quick search for a blogging solution I was confronted with the following scenarios:

  1. Build it from scratch.
    • Rails? Some JS Stack?
    • Development and Design, too time consuming for my simple needs.
  2. Wordpress.
    • Don't need all the bells and whistles.
    • I really dislike PHP my apologies!
  3. Jekyll.
    • It's great, It's simple, It's quick, It's free.
    • Don't want to waste time designing it's default blandness.
    • Would like a simple admin interface to add posts without pushing posts to github.
  4. Octopress.
    • A decked out Jekyll setup with some extra goodies.
    • Currently being rewritten in v3.0 as of writing this article.
  5. Ghost. *winner*
    • Convieant test run of service https://ghost.org/
    • Beautiful administration.
    • Easy to host and setup!
    • Many themes available and looks great from the get go!
    • Been getting into javascript development and it is powered by node.js, handlebars

To buy or deploy?

First on the list: go to https://ghost.org/ and test out the service and make sure it is something that interests you.

You have 2 options here:

  1. Decide whether you want to pay for Ghost Pro self hosting straight from the developers of the software and not deal with the extremely simple local development setup. https://ghost.org/pricing/
  2. Download the source code of the project and deploy it yourself. I chose this option or this blog post would be over.

Getting Started

  • Head over to https://ghost.org/download/ and download the ghost zip package.
  • Unzip the file.

Let's move your new blog into it's new home on your computer.

  • Open up your terminal app or iTerm (a much prefered terminal replacement).
  • We are moving the unzipped ghost folder to a Sites folder that lives in your home directory. This folder can be re-labeled to Sandbox, Dev, Sites, whatever your preference for storing development projects
  • Make sure your version number is updated to the current ghost release.
  • I am labeleling the folder that contains the development files of ghost as "myblog" you can rename this to anything you want.
mv ~/Downloads/ghost-0.5.8 ~/Sites/myblog
  • Now go into your project folder to setup your blog by building the node_modules necessary to run.
cd ~/Sites/myblog  

npm install --production

Why use npm install –production?

We have been asked a few times why, if Ghost starts in development mode by default, does the installation documentation say to run npm install --production? This is a good question! If you don’t include --production when installing Ghost, nothing bad will happen, but it will install a tonne of extra packages which are only useful for people who want to develop Ghost core itself. This also requires that you have one particular package, grunt-cli installed globally, which has to be done with npm install -g grunt-cli, it’s an extra step and it’s not needed if you just want to run Ghost as a blog.

npm start

That's it! You are up and running.

  • Navigate to http://localhost:2368 in your browser of choice (*Chrome) to view your new blog.
  • Follow the setup steps for creating your admin user.

Navigating Ghost

  1. Blog: http://localhost:2368
  2. Blog Administration: http://localhost:2368/ghost

Theming Ghost

Ghost comes with a default theme called "Casper". It is nice but, many options exist out there for adding a special touch to your new blog.

How to install a theme:

The process to install ghost themes is pretty straight foreward and universal. I will be using white fogg bootstrap responsive theme at the time of writing as an example. https://github.com/mobeenabdullah/white-fogg-ghost-theme

  • cd into your blog's theme directory.
  • Either download the zip from the projects repo, unzip it and drop the folder into the themes directory or Git clone the theme directly into the folder.
cd ~/Sites/myblog/content/themes  

git glone https://github.com/mobeenabdullah/white-fogg-ghost-theme
  • Go to http://localhost:2368/ghost and select settings from the top menu.
  • At the bottom of the page their should be an option to select your theme.

Alert: Don't delete a theme from your projects folder while it is still selected in the ghost administration. First switch to a different theme before removing a theme from your project.

Plugins & Customizations:

All of the theme's css and js files live inside the {blog-folder}/content/theme/{your-theme}/assets folder. It can be changed and customized to your needs.

For Example:

I want to have some fancy code blocks when showing example code in my blog. Let's use PrismJS.

  1. Go to http://prismjs.com/ and click Download.
  2. Select the languages you will be using most often and any additional settings like line numbers, and dark theme (okaidia) etc...
  3. Download the JS file into your theme's js folder directly.
    • ie. ~/Sites/myblog/content/themes/white-fogg-ghost-theme/assets/js
  4. Download the CSS file into your theme's css folder directly.
    • ie. ~/Sites/myblog/content/themes/white-fogg-thost-theme/assets/css
  5. Open up your blog in your development editor of choice (https://atom.io/)
    • In Terminal/iTerm open your project atom ~/Sites/myblog
  6. Open up the default.hbs file within your theme's folder and add the css and js.
    • <link href="{{asset "css/prism.css"}}" rel="stylesheet">
    • <script src="{{asset "js/prism.js"}}"></script>
  7. That's it! You now get nicely designed code blocks in your blog posts.

Deployment

To Deploy/Host your own blog, there is a list of prior requisites.

  • You need to purchase a domain for your blog. *if you do not have one
    • Go to namecheap.com and find a domain name {example}.{com}
  • You need somewhere to host it.
    • There are paid options like https://www.digitalocean.com/
    • There is free options like:
      Heroku
      Github Pages (with buster)
      OpenShift
  • Sticking with the all free motif (*domain sold separately) and for being a tad bit lazy I will be deploying my blog to github's sweet & simple pages service with the added help of buster a brute force static site generator made for ghost.

And to do just that I direct you to http://www.sitepoint.com/create-static-ghost-blog-buster/ to finish the deployment of your blog.
